# 项目介绍
本实验包含两个独立实现版本：
Python + Taichi：手动实现模型 / 视图 / 投影（MVP）矩阵，渲染旋转三角形
C + OpenGL/GLUT：固定管线渲染，实现3D 透视旋转线框立方体

# 实验一：
## 核心功能拆解
1. 技术基础与环境初始化
使用 Taichi 框架（CPU 后端）实现并行化的数值计算，Taichi 特有的 ti.field/ti.func/ti.kernel 机制确保计算效率。

2.声明两个核心 Field：vertices（存储 3D 三角形顶点）、screen_coords（存储映射后的 2D 屏幕坐标）。

3. 3D 坐标变换核心矩阵实现
整个项目的核心是计算机图形学中的 MVP 变换（Model-View-Projection），通过三个关键矩阵完成从 3D 空间到 2D 屏幕的映射：

模型矩阵（Model）：get_model_matrix 实现绕 Z 轴的旋转变换，输入角度（角度制）转换为弧度后计算旋转矩阵。
视图矩阵（View）：get_view_matrix 实现相机视角变换，将相机位置（eye_pos）移动到坐标原点，模拟 “相机看向物体” 的效果。
投影矩阵（Projection）：get_projection_matrix 实现透视投影（最贴近人眼视觉的投影方式），分为两步：
透视挤压矩阵（M_p2o）：将透视平截头体（视锥）转换为正交长方体；
正交投影矩阵（M_ortho）：将长方体缩放到标准设备坐标（NDC）的 [-1,1]³ 空间；
最终组合为完整的透视投影矩阵。

4. 并行计算与坐标映射
compute_transform 内核函数（Taichi 并行计算单元）：
组合 MVP 矩阵（遵循 “右乘原则”：proj @ view @ model）；
对每个顶点补全齐次坐标（3D → 4D），通过 MVP 矩阵计算裁剪空间坐标；
透视除法（除以齐次坐标 w）得到 NDC 坐标（[-1,1]²）；
视口变换：将 NDC 坐标映射到 GUI 窗口的 [0,1]×[0,1] 空间，存入 screen_coords。

5. 交互式可视化
初始化一个边长为 4（X/Y 轴范围 ±2）、Z 轴坐标为 -2 的正三角形顶点；
创建 700×700 的 GUI 窗口，通过键盘交互控制旋转：
按 A 键：三角形顺时针旋转（角度 +10°）；
按 D 键：三角形逆时针旋转（角度 -10°）；
按 ESC 键：退出程序；
逐帧绘制三角形的三条边（红 / 绿 / 蓝三色区分），实时展示旋转效果。

## 运行效果

运行程序后会弹出一个窗口，显示一个彩色边框的三角形，按 A/D 键可看到三角形绕中心（Z 轴）旋转，且因透视投影的存在，旋转过程中会呈现 “近大远小” 的视觉效果，符合真实 3D 场景的视觉规律。

# 实验二：C + OpenGL 3D 旋转立方体

1. 立方体几何定义
中心：原点 (0,0,0)
边长：2
顶点：8 个
边：12 条

2. 3D 透视设置
透视投影：45° FOV
相机位置：(3.5, 3.5, 3.5)
自动适配窗口比例

3. 自动旋转动画
双轴旋转（X + Y）
60 FPS 平滑动画
角度自动归环，防止溢出

4. 渲染优化
开启深度测试
加粗线条（2px）
青色高亮 + 灰色背景
双缓冲无闪烁
